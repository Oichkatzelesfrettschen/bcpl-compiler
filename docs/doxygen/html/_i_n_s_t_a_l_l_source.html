3This is an x86 (IA-32) port of an old BCPL compiler (see README)
4and should build cleanly on x86 FreeBSD and Linux. Differences
5between Linux and FreeBSD versions of the compiler are minor and
6are restricted to the system interface file "sys.s" of the compiler
7runtime, which is provided in "sys_freebsd.s" and "sys_linux.s"
8variants.
9
10The compiler sources, together with object files needed to bootstrap
11the compiler, are in the "src" directory. The "tools" directory
12contains some utility programs, and one in particular, "cmpltest.bcpl",
13will be found useful should you wish to test the compiler once that
14has been built and installed. The "docs" directory contains
15relevant documentation.
16
17To build and install the compiler in automated fashion (assuming
18you're on a supported platform), try
19
20 ./scripts/makeall.sh
21
22followed by (as root)
23
24 ./scripts/makeall.sh install
25
26This project now mandates Clang as the C compiler and makes use of the
27C23 language standard. Ensure `clang` is installed and on your `PATH`
28before building.
29
30The build defaults to a 64-bit runtime. Use ``BITS=32`` with
31``scripts/makeall.sh`` or ``make`` in ``src`` (or ``-DBITS=32`` when using CMake)
32to create 32-bit binaries or ``BITS=16`` for IA-16 targets. Set
33``CROSS_PREFIX`` as needed for cross builds (for example
34``CROSS_PREFIX=i686-linux-gnu-`` or ``CROSS_PREFIX=ia16-elf-``). ``WORDSZ`` is
35derived from ``BITS`` so a 64-bit runtime uses 8-byte words, a 32-bit
36runtime uses 4-byte words and a 16-bit runtime uses 2-byte words.
37
38Otherwise, change to the "src" directory. Check that the "sys.s"
39symbolic link points to either "sys_linux.s" or "sys_freebsd.s",
40as appropriate for your system. Enter
41
42 make
43
44to build the compiler, and
45
46 make install
47
48to install the components. By default, these mostly go in
49"/usr/local/lib/bcplc". You can also change to the "docs" directory
50and
51
52 make install
53
54to install documentation to "/usr/local/share/doc/bcplc".
55
56Should you wish to install the compiler somewhere other than
57"/usr/local/lib/bcplc", be sure to change occurrences of this path
58in the compiler sources.
59
60Once the compiler is built you can run the "cmpltest" utility from the
61"tools" directory to verify the build. If ``bcplc`` is not yet installed,
62use the driver in ``src/``. It automatically finds the runtime files in
63that directory:
64
65 make -C tools BC=../src/bcplc test
66
67After installation simply run ``make test`` in ``tools``. The test
68should conclude:
69
70 119 TESTS COMPLETED, 0 FAILURE(S)
