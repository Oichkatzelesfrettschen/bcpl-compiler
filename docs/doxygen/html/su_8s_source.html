11 * ## Architecture Support
12 * The startup code supports both 32-bit and 64-bit x86 architectures
13 * through conditional assembly based on the BITS preprocessor variable.
14 *
15 * ## BCPL Runtime Environment
16 * The startup code establishes:
17 * - Global vector for inter-module communication
18 * - BCPL stack separate from the system stack
19 * - Command-line argument processing
20 * - Standard global variables (ARGC, ARGV, etc.)
21 *
22 * ## Memory Layout
23 * - System stack: Used for system calls and startup
24 * - BCPL stack: Separate stack for BCPL program execution
25 * - Global vector: Fixed-size array for global variables
26 * - Heap: Managed separately by the BCPL runtime
